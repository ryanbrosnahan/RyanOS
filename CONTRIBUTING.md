# Contributing

RyanOS welcomes issues, ideas, documentation improvements, and pull requests.
The project is source-available under the Functional Source License 1.1 with an
Apache 2.0 future license.

irrevocable, royalty-free, sublicensable license to use, copy, modify, publish,
distribute, create derivative works from, and relicense your contribution as

This grant lets RyanOS accept community contributions while preserving the

future license changes. Your contribution may also become available under the
Apache 2.0 future license according to [LICENSE.md](./LICENSE.md).

For substantial contributions, the maintainer may require a signed contributor
license agreement before merging.

## Ground Rules

- Do not include secrets, private data, credentials, customer data, or personal
  deployment configuration.
- Do not submit code copied from another project unless the license is
  compatible and attribution is included.
- Keep generic engine code separate from private skills, private data, and
  personal configuration.
- Open an issue before large changes that affect architecture, licensing,
  security boundaries, or public APIs.

## Security

RyanOS handles personal context, credentials, notifications, and agent actions.
Do not file public issues for vulnerabilities that expose secrets, bypass
permissions, enable prompt-injection escalation, or allow unintended external
actions. Contact Ryan Brosnahan privately through GitHub first.
